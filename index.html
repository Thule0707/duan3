Du an 3

Trường hợp 1:

git init: khởi tạo

git add .: add toàn bộ dự án vào trong staging area (vùng chờ commit xác nhận)

git add ten_thu_muc: add lẻ thư mục, tệp

git commit -m "first commit": sau khi commit, code đẩy vào kho lưu trữ và đẩy lên github

git branch -M main: chuyển từ master(máy chủ git) sang nhánh main

git remote add origin https//github.com/.../...:remote đến máy chủ git

git push -u origin main: đẩy code lên máy chủ github

Trường hợp 2:

Dùng lệnh git status để ktra trước để xem đoạn code có bị sủa thêm gì không

Sau dó tiếp tục dùng lệnh git add . -> git commit -> git push

Trường hợp 4: file đã được add bị sửa đổi (có thêm thư mục hoặc file mới chưa add)

Ktra xem có file nào mới hoặc bị sửa đổi không

Sau dó dùng lệnh git add ten_thu_muc: để add file mới

Lưu ý nếu không add file sẽ giữu nguyên file cũ. Khi thêm file mới mà ko add nó sẽ không hiện trên github

Hoặc khi file đầu được add mà có sửa rồi sau đó có thêm file thứ 2 mà chưa add thì có thể dùng git add . để add tất cả file sẽ nhanh hơn

Sau đó vẫn như trên git add . hoặc git add ten_thu_muc -> git commit -> git push
